# Agoda Database Management System

## Overview
This project is a database management system inspired by the Agoda hotel booking platform. It is designed to simulate how a real-world hotel reservation system stores, manages, and retrieves data efficiently using relational database concepts.

The system focuses on handling customers, hotels, rooms, bookings, and reviews through structured database design and SQL queries.

---

## Objectives
- Design a structured relational database for a hotel booking system
- Manage customer, hotel, room, and booking data efficiently
- Perform SQL queries for data retrieval and analysis
- Apply normalization principles to reduce redundancy
- Practice real-world database system design concepts

---

## Database Features
- Customer registration and management
- Hotel and room information storage
- Booking and reservation tracking
- Review and rating system
- Data integrity using constraints (PK, FK, etc.)

---

## Technologies Used
- Oracle Database / SQL
- SQL 
- ERD (Entity Relationship Diagram)
- Database normalization (up to 3NF)

---

## 🗃️ Database Design
The system includes the following main entities:

- Customer
- Staff
- Hotel
- Room
- Booking
- Review

Relationships are established using primary and foreign keys to maintain data consistency.

---

## SQL Features Implemented
- Table creation (CREATE TABLE)
- Data insertion (INSERT)
- Data update and deletion (UPDATE / DELETE)
- Complex queries with JOIN
- Aggregation functions (COUNT, AVG, etc.)
- Constraints (PRIMARY KEY, FOREIGN KEY, NOT NULL)
